# Plan 6（改3）: 背景機能拡張とプレミアムAI共通回数設計の実装計画

## 1. 現状調査サマリー
- **背景テンプレUI / 背景適用処理:** 現在の `BgRemover.tsx` では、`templates` 配列に定義された単色や画像（グラデーション、レンガ等）を Canvas に描画する形で背景を合成しています（`applyTemplate` 関数）。ユーザー自身が画像をアップロードして背景にする「任意背景」機能は未実装です。
- **プレミアムAI利用枠（共通月30回）:** `src/lib/premiumUsage.ts` にて `PREMIUM_AI_MONTHLY_LIMIT = 30` として実装済みです。これは月ごとの `usedCount` としてカウントされ、API利用時にチェックされる基盤が整っています。

## 2. 背景機能の再整理
| 機能 | 対象 | 消費回数 | 概要・役割 |
|---|---|---|---|
| **テンプレ背景** | 全員 (Free/Pro) | なし | 用意された単色・画像パターンからクリックで背景を選択 |
| **画像を背景にする** | Pro | なし | スマホやPCから任意の画像をアップロードし、背景として合成 |
| **背景を自然になじませる** | Pro | 1回/枚 | 前景（切り抜き結果）と背景（任意/テンプレ）の光源・影をAIで自然に調和させる |
| **背景を作る** | Pro | 1回/枚 | 用意された簡単なスタイル（例: カフェ、自然、スタジオ等）＋テキストで背景をAI生成して合成 |

## 3. UI設計案
- **タブ/セクション分け:**
  背景エリアを「カラー・テンプレ（無料）」「画像を背景（Pro）」「AIで作る・なじませる（Pro/回数制限）」に整理します。
- **「画像を背景にする」:**
  ファイルアップローダー（またはカメラ起動）ボタンを配置し、選択された画像を背景として描画します。
- **「背景を自然になじませる」:**
  任意背景やテンプレ背景が選ばれた状態で表示されるトグルまたはボタン。「AIで自然になじませる（残り〇回）」のように、専門用語を省いた表現を使います。
- **「背景を作る」:**
  「カフェ風」「自然」「スタジオ」といったプリセットボタン（ポチポチ選ぶだけ）を用意し、必要に応じて「さらに詳しく（任意）」というテキスト入力欄（プロンプトにあたるもの）を設けます。
- **UI上でのガード:**
  残り回数が0の場合はボタンを非活性化し、「今月のAI機能（30回）を使い切りました」といったツールチップを表示します。

## 4. プレミアムAI回数設計
- **共通30回枠の持ち方:**
  既存の `getPremiumUsage`, `consumePremiumUsage` をそのまま利用します。
- **1回消費の対象:**
  「背景を作る（生成＋合成）」の実行で1回。
  「背景を自然になじませる（Relighting / Blend）」の実行で1回。
- **残回数表示・消費タイミング:**
  UI上部または背景機能パネル内に「AI機能: 今月残り〇回」と表示。「クレジット」という言葉は使わず、回数として表現します。サーバーで処理が成功（画像が返却されたタイミング）で消費を確定します。

## 5. `bria/generate-background` の統合方針
- **入力 / 出力:**
  元の前景画像、前景のマスク（透過部分のアルファ値）、プロンプト（UIのプリセット＋テキストを結合したもの）をAPIに送信し、完成した合成画像を受け取ります。
- **プロンプト入力のUI:**
  「どんな背景にしますか？」という問いに対し、「テーブルの上」「窓際」などのボタンを用意します。プロンプトという用語は使いません。
- **被写体保持:**
  Briaの仕様に則り、被写体の輪郭やサイズ感を保ったまま背景側を生成するモードを使用します。
- **失敗時UX:**
  「AI処理が混み合っているため、もう一度お試しください。回数は消費されません」と表示し、ステータスをエラーに戻します。

## 6. 背景を自然になじませる機能の設計
- **実行タイミング:**
  ユーザーが画像をアップロードするかテンプレを選んだ後、「自然になじませる」ボタンを押したタイミングでAPIを叩きます。
- **カバー範囲:**
  任意背景のアップロード画像に対して利用されることを主目的とします（生成背景は元からなじむため不要、または一括で提供）。
- **ボタンの意味の伝達:**
  「影や明るさを調整して、写真に自然になじませます」という補足テキストを配置します。

## 7. サーバー/権限設計
- **API Guard:**
  `/api/ai/background` エンドポイント等を作成し、リクエスト時に `getSession()` => `getPremiumUsage()` を呼びます。
- **エラーハンドリング:**
  `{ ok: false, reason: 'no_remaining' }` の場合は 403 または 402 を返し、クライアント側で「回数不足」のトーストを出します。Proでない場合は「Proの登録が必要です」と表示。

## 8. 計測 (Analytics)
以下のイベントを `trackAnalyticsEvent` 等で実装します。
1. `bg_custom_image_applied` (任意背景利用率)
2. `bg_blend_applied` (自然合成利用率)
3. `bg_generate_applied` (背景生成利用率)
4. `premium_ai_consumed` (プレミアムAI消費回数、機能名も付加)
5. `pro_purchase_click_from_ai_bg` (購入導線クリック率)
