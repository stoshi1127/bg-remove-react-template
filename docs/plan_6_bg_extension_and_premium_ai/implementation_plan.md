# Plan 6 改修：AI背景機能のUI/フロー再設計

## 概要

現在「背景除去後にAI背景生成」という2段階フローになっている部分を、**「事前に背景オプションを設定 → 一括処理」**に変更する。`bria/generate-background` モデル1つで背景除去+背景生成を一貫して処理する。

## 設計方針

### 現在のフロー
1. ユーザーが画像をアップロード
2. `handleRemove` → `/api/remove-bg`（851-labs/background-remover等）で背景除去
3. 完成後に「AIで背景を作る」「なじませる」を後付け実行

### 改修後のフロー
1. ユーザーが画像をアップロード
2. **背景カスタマイズ（オプション）内で「AIで背景を作る」or「自然になじませる」を事前設定**
3. 「処理開始」ボタンで一括処理：
   - 通常テンプレ/カラー/なし（なじませるOFF） → 従来通り `/api/remove-bg` → クライアントでテンプレ合成
   - **AIで背景を作る選択時** → `/api/ai/generate-background`（mode=generate）のみで処理（bg-removeモデル不使用）
   - **自然になじませる選択時** → `/api/ai/generate-background`（mode=blend, ref_image_file=選択中の背景）のみで処理（bg-removeモデル不使用）
4. 「自然になじませる」はテンプレ/カラー/カスタム画像選択時に表示されるチェックボックス（**デフォルトOFF**、Pro限定）

---

## 提案する変更

> [!WARNING]
> 処理フローが大きく変わるため、既存の「背景除去→後からAI」のコードパスは残し、新フローと共存させる形で段階的に実装します。

### BgRemover.tsx — State / ロジック

#### 新規 state
- `bgMode: 'normal' | 'ai_generate'` — 現在の背景処理モード  
  - `normal`: 従来通りの背景除去 + テンプレ/カラー合成
  - `ai_generate`: bria/generate-background で一括処理
- `blendEnabled: boolean` — 「自然になじませる」ON/OFF（**デフォルト: OFF**）

#### `handleRemove` の分岐追加
- `bgMode === 'ai_generate'` の場合:
  - `/api/remove-bg` ではなく `/api/ai/generate-background`（mode=generate）を呼ぶ
  - 全ファイルに対して適用し、**ファイル数 × 1回**分のプレミアムAI回数を消費
  - 処理前に残回数 ≥ ファイル数 を確認
- `bgMode === 'normal'` かつ `blendEnabled && selectedTemplate` の場合:
  - **`/api/remove-bg` は使わず** `/api/ai/generate-background`（mode=blend, ref_image_file=選択中の背景）を呼ぶ
  - bria モデルが被写体を保持しつつ背景を参照画像に合わせて再生成する
  - こちらも全ファイル × 1回消費

### BgRemover.tsx — UI

#### 背景カスタマイズ（オプション）セクションの改修

```
┌─────────────────────────────────────────────────┐
│ 背景をカスタマイズ（オプション）                    │
├─────────────────────────────────────────────────┤
│                                                 │
│ [なし] [カスタム色] [テンプレ1] [テンプレ2] ...     │
│ [画像を背景に(Pro)]                               │
│                                                 │
│ ┌─── ✨ AIで背景を作る (Pro) ─────────────────┐  │
│ │ [☕カフェ] [🌿自然] [📸スタジオ] [💼ビジネス] │  │
│ │ [🏢都市] [✈️旅行]                            │  │
│ │ [さらに詳しく（任意）__________________ ]    │  │
│ └────────────────────────────────────────────┘  │
│                                                 │
│ ☐ 自然になじませる（影・光を自動調整）[Pro]       │
│   ↑ テンプレ/カラー/カスタム画像選択時のみ表示      │
│   ↑ デフォルトOFF（ユーザーが明示的にONにする）     │
│                                                 │
└─────────────────────────────────────────────────┘
```

- 「AIで背景を作る」 は目立つカード（紫グラデーション枠）としてテンプレグリッドの直下に配置
- AIプリセットを選ぶ or テキスト入力すると `bgMode = 'ai_generate'` に切り替わる
- 通常テンプレ/カラー/なしを選ぶと `bgMode = 'normal'` に戻る
- 「自然になじませる」チェックボックスは**「AIで背景を作る」以外の背景選択時**に表示
- Pro会員の場合はデフォルトON

#### 処理開始ボタン
- `bgMode === 'ai_generate'` 時、ボタンに「AIで背景を作る（○枚 × 1回）」表示
- 残回数不足時は処理開始不可 + 案内メッセージ

### API — `/api/ai/generate-background`
- 変更なし（既存の generate/blend モード対応で十分）
- フロントから `mode: 'generate'` + `bg_prompt` or `mode: 'blend'` + `ref_image_file` を呼び分ける

## 削除するコード
- `handleAiBgGenerate` — 後付けAI背景生成ハンドラ（統合のため不要に）
- `handleBlend` — 後付けblendハンドラ（`handleRemove`に統合）
- AI背景生成の「完了後トリガー」UI（L2271-2349の完了ファイル依存セクション）
- なじませるの「完了後トリガー」UI（L2352-2390の完了ファイル依存セクション）

## 検証計画

### テスト項目
1. 通常背景（テンプレ/カラー/なし、なじませるOFF）で処理 → 従来通り動作
2. AI背景で処理（プリセット選択）→ bria/generate-background（mode=generate）で全ファイル処理
3. テンプレ + なじませるON → bria/generate-background（mode=blend）で全ファイル処理（remove-bg不使用）
4. 複数ファイル → AI回数が枚数分消費される
5. 残回数 < ファイル数 → ブロック + メッセージ
6. Free ユーザー → Pro CTA 表示
